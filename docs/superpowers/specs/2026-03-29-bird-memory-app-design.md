# 鳥記憶アプリ（とり覚え）設計仕様書

## 概要

鳥の写真を見て名前を覚える記憶術Webアプリ（PWA）。
iPhone Safariのホーム画面追加で"アプリっぽく"使える。
v1は身近な鳥約20種でプロトタイプ、将来的に花・昆虫など2000項目規模に拡張予定。

## ターゲット

- 個人利用（自分専用）
- iPhone Safari メイン
- PCブラウザでも動作

## 画面構成

### 1. ホーム画面

- アプリ名「とり覚え」＋アイコン
- 「記憶モード」ボタン → 記憶モードへ
- 「テストモード」ボタン → テストモードへ
- 進捗サマリー（登録数・覚えた数・正答率・プログレスバー）
- 設定ボタン

### 2. 記憶モード

- 写真＋名前をグリッド表示（3列）
- タップで拡大表示（写真大＋名前）
- 「想起」ボタンでテストモードへ遷移
- グリッド/1枚ずつ切り替え可能

### 3. テストモード

#### グリッド形式
- 写真をグリッド表示、名前は黒塗り（非表示）
- 各写真の下に入力欄
- 「提出する」ボタンで一括採点
- 正解=緑表示、不正解=赤表示＋正答を表示

#### 1枚ずつ形式
- 写真1枚を大きく表示
- 進捗表示（例：3/10問）
- 入力欄＋「回答」ボタン
- 即座に正誤判定、次の問題へ
- 前へ/次へボタン

### 4. 設定画面

- **出題数**: 数値を手入力（デフォルト10）
- **表示形式**: グリッド / 1枚ずつ 切り替え
- **覚えた判定**: 連続正解回数（1/2/3/5から選択、デフォルト3）
- **覚えた鳥も出題する**: ON/OFF トグル（デフォルトOFF）
- **間違えた鳥を優先出題**: ON/OFF トグル（デフォルトON）
- **エクスポート**: 進捗データをJSONファイルとしてダウンロード
- **インポート**: JSONファイルから進捗を復元
- **全データリセット**: 全進捗を削除（確認ダイアログあり）

### 5. 採点結果画面

- テスト終了後に表示
- 正答数/出題数
- 正答率
- 間違えた鳥の一覧（写真＋正答）
- 「もう一度」「ホームへ」ボタン

## データ構造

### 鳥データ（アプリ内JSON埋め込み）

```json
{
  "id": "suzume",
  "name": "スズメ",
  "category": "鳥",
  "image": "images/suzume.jpg",
  "tags": []
}
```

- `category`: 将来の拡張用（鳥/花/昆虫）
- `tags`: 将来の拡張用（季節、生息地など）
- `image`: ローカル同梱画像のパス

### 進捗データ（IndexedDB保存）

```json
{
  "id": "suzume",
  "correctStreak": 2,
  "totalCorrect": 5,
  "totalAttempts": 8,
  "mastered": false,
  "lastAttempt": "2026-03-29T12:00:00Z"
}
```

- `correctStreak`: 現在の連続正解数
- `mastered`: 連続正解数が設定値に達したらtrue
- 不正解時に`correctStreak`は0にリセット

### 設定データ（localStorage保存）

```json
{
  "questionCount": 10,
  "displayMode": "grid",
  "masteryThreshold": 3,
  "includeMastered": false,
  "prioritizeWrong": true
}
```

## 出題ロジック

1. 全鳥データから出題候補を絞る
   - `includeMastered`がOFFなら、`mastered: true`の鳥を除外
2. `prioritizeWrong`がONなら、正答率が低い鳥を優先
3. 候補からランダムに`questionCount`問を選択
4. 候補が`questionCount`より少ない場合は候補全部を出題

## 回答判定

- カタカナ完全一致のみ正解
- 全角/半角カタカナは正規化して比較
- 正解 → `correctStreak` +1、`totalCorrect` +1
- 不正解 → `correctStreak` を0にリセット
- `correctStreak` が `masteryThreshold` に達したら `mastered: true`

## 技術構成

- **HTML + CSS + JavaScript**（フレームワークなし）
- **IndexedDB**: 進捗データ保存
- **localStorage**: 設定保存
- **PWA**: manifest.json + service worker でホーム画面追加対応
- **ダークモード固定**（背景#1a1a2e系）
- **レスポンシブ**: iPhone縦画面前提
